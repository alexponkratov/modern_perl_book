=encoding utf-8

=head1 Perldoc

Z<perldoc>

X<perldoc>

Perl обладает культурой полезной документации. Утилита C<perldoc> E<mdash> часть
любой полной инсталляции Perl 5N<Однако, ваша Unix-подобная система может
потребовать установки дополнительного пакета, такого как C<perl-doc> в Debian
или Ubuntu GNU/Linux.>. C<perldoc> отображает документацию для каждого
установленного в системе Perl-модуля E<mdash> будь это базовый модуль или
установленный из CPAN (Comprehensive Perl Archive Network, Всеобъемлющая архивная
сеть Perl) E<mdash> а также тысячи страниц обширной встроенной документации
Perl.

=begin sidebar

X<CPAN>
X<metacpan>

На сайте U<http://perldoc.perl.org/> размещены свежие версии документации Perl.
Индексы CPAN на U<http://search.cpan.org/> и U<http://metacpan.org/>
предоставляют документацию по всем модулям CPAN. Другие дистрибутивы Perl 5,
такие как ActivePerl и Strawberry Perl, предоставляют локальную документацию
в формате HTML.

=end sidebar

Используйте C<perldoc> для чтения документации по модулю или части базовой
документации:

=begin screen

    $ B<perldoc List::Util>
    $ B<perldoc perltoc>
    $ B<perldoc Moose::Manual>

=end screen

Первый пример отображает документацию, встроенную в модуль C<List::Util>. Второй
пример отображает отдельный файл документации, в данном случае E<mdash> оглавление
базовой документации. Третий пример отображает отдельный файл документации,
являющийся частью CPAN-дистрибутива (L<moose>). C<perldoc> скрывает эти детали;
нет никакой разницы между чтением документации по базовой библиотеке, такой
как C<Data::Dumper>, или по установленой из CPAN.

Стандартный шаблон документации включает описание модуля, демонстрацию примеров
использования, и затем E<mdash> детальное разъяснение модуля и его интерфейса.
Объём документации различается у разных авторов, но её форма поразительно
последовательна.

=begin tip Как читать документацию

C<perldoc perltoc> выводит оглавление базовой документации, а C<perldoc
perlfaq> E<mdash> оглавление часто задаваемых вопросов (Frequently Asked
Questions, FAQ) о Perl 5. C<perldoc perlop> и C<perldoc perlsyn> документируют
символьные операторы и синтаксические конструкции Perl. C<perldoc perldiag>
объясняет значение предупреждающих сообщений Perl 5. C<perldoc perlvar>
содержит список всех символьных переменных Perl. Ознакомление с этими файлами
даст вам отличный общий обзор Perl 5.

=end tip

X<perldoc; C<-q> (search perlfaq)>

Утилита C<perldoc> имеет много других возможностей (см. C<perldoc perldoc>).
Опция C<-q> осуществляет поиск указанных ключевых слов по Perl FAQ. Так
C<perldoc -q sort> возвращает три вопроса: I<How do I sort an array by
(anything)?> (I<Как мне отсортировать массив по (чему бы то ни было)?>), I<How do
I sort a hash (optionally by value instead of key)?> (I<Как мне отсортировать
хеш (опционально по значению вместо ключа)?>), и I<How can I always keep my hash
sorted?> (I<Как мне всегда поддерживать мой хеш отсортированным?>).

X<perldoc; C<-f> (search perlfunc)>

Опция C<-f> выводит документацию по встроенным функциям Perl. C<perldoc -f sort>
объясняет поведение оператора C<sort>. Если вы не знаете имени нужной вам
функции, просмотрите список доступных встроенных функций в C<perldoc perlfunc>.

X<perldoc; C<-v> (search perlvar)>

Опция C<-v> ищет встроенную переменную. Например, C<perldoc -v $PID> выводит
документацию по переменной, содержащей идентификатор процесса текущей программы.
В зависимости от вашей оболочки, вам, возможно, придётся соответствующим образом
заключить переменную в кавычки.

Опция C<-l> заставляет C<perldoc> вывести I<путь> к файлу документации вместо его
содержимогоN<Имейте ввиду, что модуль может иметь отдельный файл F<.pod> в
придачу к файлу F<.pm>>.

Опция C<-m> отображает полное I<содержимое> модуля, код и всё остальное, не
выполняя никакого специального форматирования.

X<POD>
X<POD; C<perldoc>>
X<POD; C<podchecker>>
X<POD; C<Pod::Webserver>>
X<CPAN; C<Pod::Webserver>>

Система документации в Perl 5 E<mdash> I<POD>, или I<Plain Old Documentation>
(I<Старая добрая документация>). C<perldoc perlpod> описывает, как работает POD.
В число других инструментов для работы с POD входят C<podchecker>, который проверяет
корректность формы вашей POD-документации, и C<Pod::Webserver>, отображающий
локальную POD-документацию в виде HTML через минимальный веб-сервер.
